# Spark 粒子编辑器（SparkEditor）使用说明

本编辑器基于 **OpenSceneGraph**、**SPARK** 和 **ImGui**，用于预览、调整并导出 SPARK 粒子系统。

---

## 1. 程序能做什么

- **加载**已有的 SPARK 粒子文件（`.spk` `.xml` 等 SPARK IO 支持的格式），在 3D 视口中实时播放并调参。
- **从 Demo 起步**：内置多套官方风格示例（爆炸、雨雪、碰撞等），选中后一键载入作为模板再改。
- **从零搭建**：在界面里新增粒子组（Group）、渲染器、发射器、更新器与插值器，拼出一条完整效果链路。
- **保存 / 导出**：将当前系统写出为 **`.spk` + `.spk.xml`** 两份文件，并把编辑器能识别的贴图复制到保存目录。

**简单原理**：SPARK 把效果拆成「系统 System → 若干组 Group」。每个组有自己的**容量**、**生命周期**、**发射器**（在哪、以多快的速率喷粒子）、**更新器**（重力、障碍等）、**插值器（颜色/参数插值）**和**渲染器**（点、线、四边形 billboard 等）。编辑器把这套结构用树状面板暴露出来，改完写回文件。

---

## 2. 启动与资源路径

- **内置Demo粒子贴图**：位于 `Resource/`（如 `explosion.bmp`、`point.bmp` 等）。
- 主窗口标题为 **「Spark Particle Editor」**；左侧或叠加的是 **ImGui 面板**，中间是 **OSG 3D 视口**（含简单坐标轴，便于辨认方向）。
- **相机**：用 OSG 查看器默认操作（一般为鼠标拖拽旋转、滚轮缩放等）。面板上会显示 **Camera Info**（位置、旋转、缩放）。

---

## 3. 界面分区说明

| 区域 | 作用 |
|------|------|
| **Real FPS / Avg FPS** | 性能参考；仿真步长与显示有关，仅供参考。 |
| **Camera Info** | 当前相机位姿数值。 |
| **Demo Preset** | 下拉选择内置 Demo 粒子名称。 |
| **Load Selected Demo** | 用当前下拉项**重新生成**整套粒子（会覆盖当前内存中的系统，未保存的修改会丢失）。 |
| **System name** | 当前粒子系统在 SPARK 内的名称（保存时可能作为默认建议名）。 |
| **Save / Load** | 保存到磁盘 / 从磁盘打开粒子文件（Windows 下为系统文件对话框）。 |
| **Particle System Tree** | 核心编辑区：按 **Group（组）** 展开，编辑渲染器、发射器、修饰器、插值器。 |
| **New group +** | 在系统末尾**新建一个空组**（带默认容量与简单发射器，便于从空白开始搭）。 |
| **Source File** | 当前编辑对应的源文件路径提示（加载或保存后会更新）。 |

## 4. 内置 Demo 预设（模板）

下拉框 **Demo Preset** 中可能包含（名称与程序内注册一致）：

- **SPKTest** — 拖尾、附着等复合效果示例。  
- **SPKCollision** — 粒子在盒内碰撞、摩擦等。  
- **SPKFlakes** — 大量雪花式点精灵。  
- **SPKExplosion** — 爆炸烟、火、闪光、冲击波等多组组合。  
- **SPKTestIrrlicht** / **SPKTestIrrlicht_Controllers** — Irrlicht 风格测试。  
- **DX9.BasicDemo**、**DX9.GravitationDemo**、**DX9.RainDemo** — 经典 DX9 Demo 的 GL 近似版（如雨滴、线雨、溅起水花等）。

**用法建议**：先选一个接近目标效果的 Demo，点 **Load Selected Demo**，再在 **Particle System Tree** 里微调，比完全从零快很多。

## 5. 加载与保存

### 5.1 加载

1. 点击 **Load**，在文件对话框中选择粒子文件（格式为 `.spk`、`.xml`）。  
2. 加载成功后，程序会尝试根据文件**所在目录**解析渲染器里记录的**贴图相对路径**，并上传到显卡。若贴图与 `.spk` 不在同一目录或路径不对，可能看不到贴图或控制台有报错，需把贴图放到与文件记录一致的位置。

### 5.2 保存

1. 点击 **Save**，选择保存位置和**主文件名**（无需自己加后缀）。  
2. 程序会生成：  
   - **`<主文件名>.spk`** — SPARK 二进制；  
   - **`<主文件名>.spk.xml`** — XML 描述，便于 diff 或工具链。  
3. 对编辑器登记过的贴图（例如来自内置 `Resource` 演示贴图，或在本工具里通过「加载纹理」选中的图），保存时会尽量把**贴图文件复制到与 `.spk` 相同的输出目录**，并在数据里写入**文件名**，方便整包交给程序：「同目录下的 `.spk` + `.spk.xml` + 若干 `.bmp`/`.png`」。

**注意**：若某张贴图从未经本工具路径登记（控制台可能提示 texture id 无注册名），保存时可能**无法**把贴图名写进资源，程序同事加载时会找不到图。自定义贴图请在 **GLQuadRenderer** 面板里用 **Load texture** 选一次文件，再保存。

## 6. 「Particle System Tree」怎么读

树按 **Group 0、Group 1…** 分组。每个组下通常包含四类节点（有则显示，无则显示 `(none)`）：

1. **Group Properties** — 本组全局：容量、寿命、半径、是否永生等。  
2. **Renderer** — 本组用什么画出来（四边形、点、线、拖尾等）。  
3. **Emitters** — 一个或多个发射器：决定粒子**从哪来**、**多快**、**初速度大致范围**。  
4. **Modifiers** — 物理/行为修饰：重力、摩擦、障碍、涡旋等（**每种类型在同一组里最多加一个**，重复类型按钮会灰掉并提示 exists）。  
5. **Interpolators** — 颜色随生命周期变化，以及 scale、mass、angle、textureIndex、rotationSpeed 等**浮点参数**的插值/随机/曲线。

各节点左侧 **「-」** 为删除（删除组会删掉整组粒子逻辑，请谨慎）。

## 7. 常用参数说明（美术向）

### 7.1 Group Properties（组属性）

- **Capacity (10^n)**：组内**粒子槽位**数量级，显示为 \(10^n\)（例如 n=3 表示约 1000 个槽）。粒子很多却「发不出来」时，首先检查容量是否够。  
- **Life Min / Life Max**：每个粒子存活时间的随机范围（秒）。  
- **Graphical Radius / Physical Radius**：分别影响**显示**与**碰撞/检测**用的粒子半径量级（具体以 SPARK 文档为准）。  
- **Immortal**：永生不会按寿命杀粒子，常用于「场景中常驻的一片雾/尘」（常与静态或循环发射配合）。  
- **Still**：粒子是否不参与某些运动更新。  
- **Sorting**：是否对粒子排序（通常与透明、叠加画法有关，乱序时可能出现叠层错误）。

### 7.2 Renderer（渲染器）

可选类型（下拉或新建时）：**GLQuadRenderer**（带纹理的片）、**GLPointRenderer**（点/点精灵）、**GLLineRenderer**、**GLLineTrailRenderer**（拖尾）。

- **Blend Mode**：常见为 **NONE / ALPHA / ADD**。加法混合适合火、光晕；Alpha 适合烟、半透明片。  
- **Alpha Test / Depth Test / Depth Write**：深度与裁切选项，与引擎里是否写入深度、是否做 Alpha 测试一致（由于opengl版本问题，Alpha Test 已无效）。  
- **GLQuadRenderer** 下可设 **Atlas X/Y**（纹理图集格子数）、**Quad Scale**、以及 **Texture**：点击预览图或 **Load texture** 从磁盘选贴图（仅四边形渲染器在面板里提供换图；点精灵等若需换图可能依赖加载文件或程序侧扩展）。

### 7.3 Emitters（发射器）

类型包括：**StaticEmitter**、**RandomEmitter**、**StraightEmitter**、**SphericEmitter**、**NormalEmitter** 等。

- **Flow**：发射速率相关（负值表示瞬间发射全部Tank数目粒子，与 **Tank Min/Max** 组合可表达「无限喷」或「喷完为止」等逻辑；Tank、Flow 不能同时为负，界面做了二者的范围限制）。  
- **Force Min/Max**：初速度或冲量随机范围。  
- **Zone (…)**：发射形状（点、球、盒、圆柱、平面、环等）。可改 **Zone Type** 切换形状；若提示 **shared zone**，表示多个对象共用同一块区域数据，改一处会影响所有引用。

### 7.4 Modifiers（修饰器）

**规则**：同一组里**每种修饰器类型只能加一个**（重复类型时 **+** 会禁用并提示 exists）。选中后在树内展开即可调该类型专有参数。

以下为编辑器里可新建的各类修饰器**简要说明**（细节以 SPARK 行为为准）：

| 类型 | 简单介绍 |
|------|----------|
| **Gravity（重力）** | 给粒子施加一个方向恒定的加速度，模拟下落、上扬等。常用向量如 `(0, -1, 0)` 表示「往下掉」。 |
| **Friction（摩擦/阻尼）** | 让速度逐渐衰减，粒子会越跑越慢，适合烟、灰尘慢慢停下，或雨水落地后减速。 |
| **Obstacle（障碍）** | 粒子碰到指定的**几何区域（Zone）**时反弹或受阻，可设反弹强度、表面摩擦等。用于地面、墙壁、球壳等「碰一下弹开或滑开」。 |
| **LinearForce（线性力场）** | 在某一空间范围内对粒子施加与位置/规则相关的力（编辑器里新建时带默认区域）。适合做「风区」「局部推力」等方向性场效。 |
| **Collider（碰撞体）** | 粒子之间的**相互碰撞**（弹性等参数可调），适合沙粒、碎屑彼此弹开，与 Obstacle（对静态几何）概念不同。 |
| **PointMass（点质量/引力点）** | 在空间某一点产生**引力或斥力**，粒子会被吸向该点或受类似轨道影响。适合做漩涡中心、黑洞口、吸引特效。 |
| **RandomForce（随机力）** | 按随机方向、随机周期对粒子施加扰动，轨迹更「乱」、更自然，适合火星飞溅、抖动烟雾等。 |
| **Vortex（涡旋）** | 绕轴旋转并可选向中心吸引，形成螺旋、旋风状运动。可配合 **Kill In Eye** 等参数在核心区销毁粒子。 |
| **EmitterAttacher（发射器附着）** | 让一个组里粒子的生成**跟随另一个发射器**的位置/朝向（常用于拖尾、子粒子跟主体）。新建后往往需在 TA 协助下把引用绑到具体发射器/组。 |
| **Destroyer（销毁器）** | 粒子进入指定 **Zone** 内即被销毁，用于「飞出屏幕就删」「落进水里消失」等边界清理。 |
| **Rotator（旋转器）** | 按规则改变粒子的**朝向角**（与贴花片的面向、自旋有关），适合做纸片翻转、火花朝向变化等。 |

### 7.5 Interpolators（插值器）

- **颜色**：支持简单双色、随机、随机双色过渡、**颜色曲线（ColorGraph）** 等。曲线类可在 **Graph Entries** 里增删控制点（按 X 排序）。  
- **浮点参数**：先选槽位（**scale / mass / angle / textureIndex / rotationSpeed**），再选插值器类型（简单、默认常数、随机、随机双端、**Float Graph** 等），与颜色类似，Graph 类型可编辑关键帧。

## 8. 从 0 到 1 搭一条简单效果（推荐流程）

1. 点击 **New group +**，得到一个新组。  
2. 在 **Renderer** 里选 **GLQuadRenderer**，点 **+**；展开后设 **Blend Mode**（例如 ADD），用 **Load texture** 指定一张 RGBA 合适的片纹理。  
3. 在 **Emitters** 里保留或添加 **RandomEmitter** / **StraightEmitter** 等，把 **Flow** 调到可见的数值，在 **Zone** 里把发射区域移到视口前方（调整 **Position** 等）。  
4. 在 **Interpolators** 里加 **ColorSimpleInterpolator** 或 **ColorGraphInterpolator**，控制从生到死的颜色变化。  
5. 按需加 **Gravity**、**Friction** 等 **Modifiers**。  
6. 用 **Save** 导出 `.spk` + `.spk.xml` 及贴图，把整包交给程序同事。

若某一步后界面树「卡住」或显示异常，可尝试再次点击 **Load Selected Demo** 或重新 **Load** 文件（会丢未保存修改）。

## 9. 已知限制

- 编辑器聚焦在 **SPARK 1.x 风格固定管线 GL 渲染路径** 与 IO，与项目最终运行时（核心模式opengl、自定义 Shader）可能仍有差异，**最终以引擎内效果为准**。  
- **EmitterAttacher** 等复杂修饰器新建时可能是默认空引用，暂不支持；  
- **SPKTestIrrlicht_Controllers** 与不带后缀的版本在编辑器里基底相同，部分 Irrlicht 控制器未暴露到 UI。
